# Security Policy

## Supported versions

Security fixes are applied to the latest released version of each ReproRelay package and to the `main` branch.

## Report a vulnerability privately

Do not open a public issue for a suspected vulnerability or include credentials, personal data, captured evidence, or exploit details in Discussions.

Use [GitHub's private vulnerability reporting](https://github.com/kish613/reprorelay/security/advisories/new). Include the affected component and version, reproduction steps, impact, and any suggested mitigation. Maintainers will acknowledge a complete report as soon as practical and coordinate remediation and disclosure with the reporter.

## Deployment responsibility

ReproRelay is self-hosted. Operators are responsible for access control, project keys, storage privacy, database security, GitHub App permissions, evidence retention, and any optional email or AI-provider credentials in their deployment.
